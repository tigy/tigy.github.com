jPlusUI 设计理念

jPlusUI 目标

  • 面向互联网页面，不面向企业级应用。
  • 可以快速搭建出漂亮、兼容的页面。
  • 轻量、稳定、简单，所有组件一行代码即现效果。

组件的定义

  • 任何单独写出来的可以重用的代码都叫组件。
  • 组件可以只有js文件，也可以只有css文件。
  • 组件之间可以互相依赖，甚至可以依赖比如 jQuery、Understone 等流行框架。

设计原则

组件一键初始化
所有 css 组件都只需加一个 class 即可见到效果。所有 js 组件都只需一行代码即可完成初始化。比如 new ComboBox("id") 。
提供常用 API，但不作完美主义
jPlusUI 只提供常用的API ，不常见的需求也能通过现有的API组合实现。比如 comb.setSelectedIndex(0) 。
HTML优先
jQueryUI 的设计原则是一切以 HTML 为主，然后通过 JavaScript 动态渲染组件。但是这样组件缺乏动态操作的特性（比如很难动态增加一个tab）。
YUI/Ext 的设计原则是一切是 JavaScript 为主，组件API 丰富，因此代码量大，学习成本高，不适合Web应用。
jPlusUI 则是取两者的优点：所有组件以HTML为主，并由JavaScript负责逻辑控制，提供常用（但不全）的API，用于动态操作。
适度的继承
继承可以减少很多代码量，但是过多的继承会导致模块过细，学习和使用困难。所以原则上继承链不超过 3 个。在有些地方，宁可复制同样的代码，也不提取公共基类。

jPlusUI 和其它框架的区别

和 jQuery/Mootools 的区别
jQuery/Mootools 的侧重点是 DOM 操作，而 jPlusUI 的侧重点是 UI。jPlusUI 内部也有 DOM 操作的模块，操作上和 jQuery 一样方便，但更重要的是它为UI组件的开发提供了底层支持：
  1. 组件本身也是DOM节点，通过继承的方式可直接调用DOM操作的函数。
  2. 组件提供的API和DOM操作的API一致，减少每个组件的学习成本。
  3. 支持组件的自定义事件，方法重写。
用户也可以使用 jQuery + jQuery-Adapter 作为UI底层。甚至可以和 jQuery 一起使用。
和 Bootstrap 的区别
jQuery 作为 DOM 操作的框架，已经非常完美了。而在 UI 上，却始终没有一个完整的、像jQuery一样方便的框架出现，直到 Bootstrap 的出现。Bootstrap 确实做的很好，jPlusUI 3.0 版本以后也从 Bootstrap 吸取了很多精华。但是 jPlusUI 仍在以下方面拥有绝对优势：
  1. jPlusUI 的组件数量远超过 Bootstrap 。且包含很多适合国情的组件（如城市选择器）。
  2. Bootstrap 显示英文更漂亮； jPlusUI 显示中文更漂亮。并且 jPlusUI 可以保证很少出现中文乱码的问题。
  3. jPlusUI 支持 IE6 ，而Bootstrap只支持到 IE7。为了支持IE6， jPlusUI 的部分API 比 Bootstrap 更长。
  4. jPlusUI 在组件class上设计上更统一、易于识别和记忆。
  5. 在 JavaScript 上，jPlusUI 使用了面向对象的设计，让组件更灵活。
和 YUI/Ext 的区别
YUI/Ext 目标在于提高完整 API 的库，它们更适合企业级的应用。而 jPlusUI 轻量小巧，更适合互联网级的应用。
