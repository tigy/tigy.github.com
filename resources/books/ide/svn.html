SVN 是一个版本控制软件。 用于代码开发时控制代码版本。比如一个文件肯能经常被修改， SVN 帮助切换到修改的任何版本。 现在有很多的 SVN 客户端软件，建议使用 eclipse 插件-- subclipse 。 SVN 常用的操作: 1. 从 SVN服务器下载并新建 项目。 以 eclipse 为例。 1. 点击 文件/项目/SVN/从 SVN 导出项目... 2. 填写 SVN 地址。 3. 下一步。 4. 选择 trunk 目录，或其它根目录。 5. 下一步。 6. 选择 作为当前工作空间的项目 导出。 7. 选择项目的名字和位置。 8. 完成。 9. 等待 SVN 下载完成。 2. 将项目添加至 SVN 服务器。 以 eclipse 为例。 1. 右击项目。 2. 选择 团队(Team)/共享项目 (Public Project...) 3. 填写 SVN 地址。 4. 完成。 3. 日常操作: 1. 提交(Submit)。 将本地代码提交到服务器。如果这份代码有其它人修改过，则会报出 OutOfDate (过期的) 错误， 这时见操作 4。 2. 更新(Update)。将服务区代码更新到本地。如果这份代码本地修改过，则不会更新覆盖。 3. 更新到某版本(Update to version)。类似更新，但目标是指定版本，不是最新版本。 4. 合并(Merge)。如果服务器的代码和本地有冲突，可以使用合并，把2份代码合成新的版本。 5. 清理(Clean)。如果SVN 缓存出现错误，可以使用它清理。 1. 登陆项目主页 2. 进入 源码( Source ) 3. 选择 检出( Checkout ) 4. 复制上面那个 https 的地址。 这就是 SVN的地址。 5. 点击 googlecode.com password. 后可以得到密码。这个网站肯能需翻墙。 SVN的地址，一般最后是 trunk/ 结尾。 每个人的密码都是不同且固定的。建议保存好这个密码。 如果 对 SVN不了解， 可以读 SVN 使用.txt
